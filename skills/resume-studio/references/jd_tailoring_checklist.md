# JD 定制检查清单（输出 changes.md 时用）

## 1. JD 解析（先做）
- 硬性要求（必须满足/必须出现的证据）
- 加分项（可选增强点）
- 工作内容主轴（3–5 条）
- 关键词簇：技术栈、业务领域、方法论、指标
- 雷区：不匹配的领域/年限/必备证书缺失

## 2. 内容映射（再做）
对每条 JD 主轴，至少给出一条简历证据（经历/项目/bullet）：
- 证据写法：动作 + 方法 + 结果（量化）+ 影响范围
- 证据不足时：写“可迁移证据”（相似问题/相似指标/相似约束）

## 3. 结构调整（常见有效手段）
- 把与 JD 最相关的经历/项目上移；不相关内容缩短或删除。
- 把“技能列表”变成“技能证据”：技能出现于做过的事里。
- 用 1–2 行 Summary/Highlights（可选）把关键词与定位集中表达。

## 4. 真实性边界（强约束）
- 不新增虚构公司、项目、数据、职位、奖项。
- 数字必须可解释口径（比如“离线评估”“A/B 实验”“样本量”）。
- 不把“接触/协作”写成“主导/负责”。

## 5. changes.md 输出格式（建议）
- 改动总览（3–8 条）
- 逐段改动：原段落意图 → 新段落意图 → 对应 JD 证据
- 风险提示（如果存在不确定、需用户确认的信息）
